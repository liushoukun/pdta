# 需求池

## 📖 说明

需求池用于管理所有待评估、已批准和已拒绝的需求，确保需求不遗漏。

## 📋 需求状态

### 待评估需求 (Pending)

[查看待评估需求列表](./pending.md)

新提出的需求，等待产品经理和技术团队评估。

### 已批准待排期 (Approved)

[查看已批准需求列表](./approved.md)

已通过评估但尚未排期开发的需求。

### 已拒绝需求 (Rejected)

[查看已拒绝需求列表](./rejected.md)

经评估后暂不实现的需求，记录拒绝原因。

## 📝 需求提交流程

1. **需求提出**：在 `pending.md` 中添加新需求
2. **需求评估**：产品经理组织评估会议
3. **状态更新**：根据评估结果移动到对应列表
4. **需求排期**：已批准需求进入迭代计划
5. **创建需求文件夹**：开始编写 PRD

## 🎯 需求评估标准

- ✅ 业务价值
- ✅ 用户需求强度
- ✅ 技术可行性
- ✅ 开发成本和时间
- ✅ 与产品规划的匹配度
