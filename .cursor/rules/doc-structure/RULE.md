# 文档结构规范

> 定义产研测一体化文档的目录结构和命名标准

## 规则概述

所有需求文档必须遵循统一的目录结构，确保AI和团队成员能够快速定位和理解文档内容。

---

## 目录结构规范

### 标准结构

```
docs/
├── 需求文档/                  # 所有需求的根目录
│   ├── {年月}-{需求名称}/      # 单个需求文件夹
│   │   ├── product/           # 产品方案目录
│   │   │   └── PRD.md        # 产品需求文档
│   │   ├── develop/           # 技术设计目录
│   │   │   └── TDD.md        # 技术设计文档
│   │   └── test/              # 测试文档目录
│   │       └── TCD.md         # 测试用例文档
│   └── ...
├── 项目说明/                  # 项目级文档
├── 用户手册/                  # 面向用户的文档
└── 开发文档/                  # 开发规范和指南
```

---

## 命名规范

### 需求文件夹命名

**格式**：`{年月}-{需求名称}`

**规则**：
- 年月：`YYYYMM` 格式（6位数字）
- 需求名称：简洁清晰，使用中文
- 分隔符：使用短横线 `-`
- 避免：特殊字符、空格、英文缩写

**示例**：
- ✅ `202602-用户登录模块`
- ✅ `202603-移动端首页改版`
- ✅ `202604-数据报表优化`
- ❌ `2026-02-用户登录` （不符合年月格式）
- ❌ `202602_user_login` （不应使用英文和下划线）
- ❌ `202602-用户 登录` （不应包含空格）

### 文档文件命名

**固定文件名**（不可修改）：
- 产品文档：`PRD.md`
- 技术文档：`TDD.md`
- 测试文档：`TCD.md`

**补充文档**（可选）：
- API文档：`API.md`
- 部署文档：`DEPLOY.md`
- 变更记录：`CHANGELOG.md`

## 子目录规范

### product/ 目录
- 存放产品相关文档
- 核心文件：PRD.md
- 可包含：原型图、需求草稿、评审记录

### develop/ 目录
- 存放技术设计文档
- 核心文件：TDD.md
- 可包含：架构图、ER图、API草稿

### test/ 目录
- 存放测试相关文档
- 核心文件：TCD.md
- 可包含：测试报告、bug记录

---

## 文件组织原则

### 1. 完整性原则
每个需求文件夹必须包含三个核心子目录（product、develop、test）

### 2. 独立性原则
每个需求独立成文件夹，互不影响

### 3. 可追溯性原则
通过文件夹名称可快速定位需求所属时间和主题

### 4. 易导航性原则
目录层级清晰，不超过3层

---

## AI行为约束

### 创建文档时
- 自动检查是否符合命名规范
- 自动创建三个必需子目录
- 自动使用标准模板初始化文档

### 引用文档时
- 使用相对路径引用同需求内的文档
- 引用其他需求时使用完整路径
- 提供文档间的清晰链接

### 搜索文档时
- 优先按年月搜索（最近的需求）
- 支持按需求名称模糊搜索
- 提供文档完整性检查

---

## 异常情况处理

### 文件夹命名不规范
❌ 发现：`202602_user_login`
✅ 修正：建议重命名为 `202602-用户登录模块`

### 缺少必需子目录
❌ 发现：只有 `product/` 目录
✅ 修正：自动创建 `develop/` 和 `test/` 目录

### 文档文件名错误
❌ 发现：`product/产品需求文档.md`
✅ 修正：重命名为 `product/PRD.md`

## 版本控制建议

### Git忽略规则
```gitignore
# 临时文件
docs/**/*.tmp
docs/**/*.bak

# 个人笔记
docs/**/*_notes.md
```

### 提交规范
- 新需求：`docs: 创建202602-用户登录模块需求`
- 更新文档：`docs: 更新202602-用户登录模块PRD`
- 完成需求：`docs: 完成202602-用户登录模块`

---

## 检查清单

在创建或修改文档结构时，确认以下内容：

- [ ] 需求文件夹命名符合 `{年月}-{需求名称}` 格式
- [ ] 包含 `product/`、`develop/`、`test/` 三个子目录
- [ ] 每个子目录包含对应的标准文档文件
- [ ] 文档文件名严格遵循命名规范
- [ ] 目录结构层级不超过3层
- [ ] 无特殊字符和空格在文件夹名称中

---

## 示例：正确的文档结构

```
docs/需求文档/202602-移动端首页改版/
├── product/
│   ├── PRD.md                    # ✅ 标准命名
│   ├── 原型图.png                 # ✅ 补充资料
│   └── 需求评审记录.md            # ✅ 补充文档
├── develop/
│   ├── TDD.md                    # ✅ 标准命名
│   ├── 架构图.mmd                # ✅ mermaid图表
│   └── API设计草稿.md            # ✅ 补充文档
└── test/
    ├── TCD.md                    # ✅ 标准命名
    └── 测试报告.md                # ✅ 补充文档
```
